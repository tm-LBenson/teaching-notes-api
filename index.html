Simple, reliable API docs.

The North Star API is a lightweight REST API built with Express, JWT authentication, and Firestore. Each account has isolated note data, making it ideal for learning authentication, protected routes, and CRUD workflows using tools like Postman or fetch.

Start in 5 minutes View examples

API highlights

Auth

Register, login, and inspect the current user with bearer tokens.

Resource model

Each note is bound to a single user through userId.

Developer friendly

CORS enabled, JSON responses, clear status codes, and predictable errors.

Overview

What this API provides

REST

Authentication

Use /auth/register and /auth/login to obtain a JWT. Send that token in the Authorization header for protected routes.

Notes CRUD

Create, list, read, update, and delete notes using the /notes resource.

Scoped data

Users only see their own records. Requests for another user’s note return not found.

Base URL

Environment endpoint

Base URL:

https://north-star-b834.onrender.com

Authentication

Bearer token flow

Step 1

Register or log in

Call an auth endpoint with an email and password to receive a token.

Step 2

Store the token

Copy the JWT returned by the API and store it in your client or API tool.

Step 3

Send Authorization header

Attach the token to protected requests using the standard bearer token format.

Authorization header

Authorization: Bearer YOUR_JWT_TOKEN

Make your first successful request

1. Register

2. Use the returned token

Response

  "token": "<jwt>"

Recommended first workflow

Register, call GET /auth/me, create a note with POST /notes, then verify it with GET /notes.

Endpoints

Reference

All responses are JSON. Protected routes require a bearer token. Example request and response bodies are shown below each endpoint.

POST

/auth/register

Create a new user account and immediately return a JWT.

Request body

{
  "email": "student@example.com",
  "password": "password123"
}

Response 201

{
  "token": "<jwt>"
}
POST

/auth/login

Verify credentials and return a JWT for the account.

Request body

{
  "email": "student@example.com",
  "password": "password123"
}

Response 200

{
  "token": "<jwt>"
}
GET

/auth/me

Auth required

Return the currently authenticated user from the JWT payload.

Headers

Authorization: Bearer YOUR_JWT_TOKEN

Response 200

{
  "id": "user_123",
  "email": "student@example.com"
}

Errors

Error format and common status codes

Standard error response

{
  "error": "title and body are required"
}

Common status codes

200 Successful read, update, or delete confirmation
201 Resource created successfully
400 Invalid request body or missing required fields
401 Missing, invalid, or expired bearer token
404 Route or note not found
429 Too many requests
500 Internal server error

Rate limits and CORS

Operational behavior

Rate limiting

The API uses a simple request limiter to protect the service and demonstrate production-style request controls.

CORS

CORS is enabled so browser-based clients can call the API from a static site such as GitHub Pages.

Code examples

Use it from fetch or curl

fetch example

const token = 'YOUR_JWT_TOKEN'
const options = {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${token}`
  },
  body: JSON.stringify({
    title: 'My first note',
    body: 'Hello from the API'
  })
}
const response = await fetch('http://localhost:3000/notes', options)

const data = await response.json()
console.log(data)

curl example

curl -X POST http://localhost:3000/notes \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "title": "My first note",
    "body": "Hello from the API"
  }'
